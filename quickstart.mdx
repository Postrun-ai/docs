Postrun is a REST API at `https://api.postrun.ai/v1`. Every request carries a
bearer credential. This page gets you a working call, then hands the browser a
scoped token so your frontend can talk to Postrun directly.

## Authentication

There are two kinds of credential, and the difference is the whole security
model:

- **Secret API key** (`pr_…`) — full account access. Use it only from your
  **backend**; never ship it to a browser.
- **Frontend token** — a short-lived, profile-scoped JWT your backend mints from
  a secret key and hands to the browser. It grants only the scopes and profiles
  you mint into it. Money and structural writes are secret-key only and can
  never be reached with a browser token.

<Steps>
  <Step title="Get a secret key">
    Create a `pr_` key in the [dashboard](https://app.postrun.ai). Keep it on
    your server — set it as an environment variable, never commit it.
  </Step>
  <Step title="Make your first request">
    List the profiles under your account. A profile is a brand or client you
    publish and run ads for.

    ```bash
    curl https://api.postrun.ai/v1/profiles \
      -H "Authorization: Bearer $POSTRUN_SECRET_KEY"
    ```

    Every response is a typed envelope. A list looks like:

    ```json
    {
      "object": "list",
      "data": [
        {
          "id": "prof_01h455vb4pex5vsknk084sn02q",
          "name": "Acme Co.",
          "external_id": "client_acme",
          "metadata": { "plan": "pro" },
          "created_at": "2026-06-16T14:32:08Z"
        }
      ],
      "total": 1,
      "limit": 20,
      "has_more": false
    }
    ```
  </Step>
  <Step title="Mint a frontend token">
    From your backend, exchange the secret key for a scoped token and send it to
    the browser. Scope it to exactly what that session needs.

    ```bash
    curl https://api.postrun.ai/v1/tokens \
      -H "Authorization: Bearer $POSTRUN_SECRET_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "profile_scope": { "type": "ids", "values": ["prof_01h455vb4pex5vsknk084sn02q"] },
        "scopes": ["posts:read", "posts:write", "media:write"]
      }'
    ```
  </Step>
  <Step title="Call directly from the frontend">
    The browser uses the token as its bearer — no proxy of your own. The same
    API, the same envelope, narrowed to the token's scopes and profiles.

    ```ts
    import { createPostrunClient } from "@postrun/js";

    const client = createPostrunClient({ getToken });
    const { data } = await client.profiles.list();
    ```
  </Step>
</Steps>

## Writes are safe to retry

Every write accepts an `Idempotency-Key` and a dry-run, so a network blip never
double-posts or double-spends:

```bash
curl https://api.postrun.ai/v1/posts \
  -H "Authorization: Bearer $POSTRUN_SECRET_KEY" \
  -H "Idempotency-Key: $(uuidgen)" \
  -H "Content-Type: application/json" \
  -d '{ "dry_run": true, "...": "..." }'
```
