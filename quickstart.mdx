Postrun is a REST API at `https://api.postrun.ai/v1` with a typed SDK,
[`@postrun/js`](https://www.npmjs.com/package/@postrun/js). Every example below
shows both. By the end you'll have made an authenticated request and minted a
scoped token your frontend can call Postrun with directly.

Two credentials do the work here: a **secret API key** (`pr_…`) with full account
access, used on your backend only; and a **scoped token**, short-lived and
limited to the profiles and scopes you grant it, safe to hand the browser. You'll
use both below — see [Authentication](/authentication) for the full model.

<Steps>
  <Step title="Get a secret key">
    Create a `pr_` key in the [dashboard](https://app.postrun.ai). Keep it on
    your server — set it as an environment variable, never commit it.
  </Step>
  <Step title="Make your first request">
    List the profiles under your account. A profile is a brand or client you
    publish and run ads for.

    <CodeGroup>
    ```ts @postrun/js
    import { createPostrunServer } from "@postrun/js/server";
    import { profilesList } from "@postrun/js";

    const postrun = createPostrunServer({
      secretKey: process.env.POSTRUN_SECRET_KEY!,
    });

    const { data } = await profilesList({ client: postrun });
    ```

    ```bash cURL
    curl https://api.postrun.ai/v1/profiles \
      -H "Authorization: Bearer $POSTRUN_SECRET_KEY"
    ```
    </CodeGroup>

    Every response is a typed envelope. A list looks like:

    ```json
    {
      "object": "list",
      "data": [
        {
          "id": "prof_01h455vb4pex5vsknk084sn02q",
          "name": "Acme Co.",
          "external_id": "client_acme",
          "metadata": { "plan": "pro" },
          "created_at": "2026-06-16T14:32:08Z"
        }
      ],
      "total": 1,
      "limit": 20,
      "has_more": false
    }
    ```
  </Step>
  <Step title="Mint a frontend token">
    From your backend, exchange the secret key for a scoped token and send it to
    the browser. Scope it to exactly what that session needs. It's short-lived —
    15 minutes by default; set [`ttl_seconds`](/authentication#lifetime) to
    change it.

    <CodeGroup>
    ```ts @postrun/js
    import { createPostrunServer } from "@postrun/js/server";

    const postrun = createPostrunServer({
      secretKey: process.env.POSTRUN_SECRET_KEY!,
    });

    const { token } = await postrun.tokens.mint({
      profile_scope: { type: "profile_ids", values: ["prof_01h455vb4pex5vsknk084sn02q"] },
      scopes: ["posts:read", "posts:write", "media:write"],
    });
    ```

    ```bash cURL
    curl https://api.postrun.ai/v1/tokens \
      -H "Authorization: Bearer $POSTRUN_SECRET_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "profile_scope": { "type": "profile_ids", "values": ["prof_01h455vb4pex5vsknk084sn02q"] },
        "scopes": ["posts:read", "posts:write", "media:write"]
      }'
    ```
    </CodeGroup>
  </Step>
  <Step title="Call directly from the frontend">
    The browser sends the token as its bearer and calls Postrun directly — no
    proxy of your own. It's the same API and the same envelope, narrowed to the
    token's scopes and profiles.

    <CodeGroup>
    ```ts @postrun/js
    import { createPostrunClient, postsList } from "@postrun/js";

    // Browser — getToken fetches the token your backend minted above.
    const postrun = createPostrunClient({ getToken });

    const { data } = await postsList({ client: postrun, query: { limit: 20 } });
    ```

    ```bash cURL
    curl https://api.postrun.ai/v1/posts \
      -H "Authorization: Bearer $POSTRUN_FRONTEND_TOKEN"
    ```
    </CodeGroup>
  </Step>
</Steps>

## Writes are safe to retry

Every write accepts an `Idempotency-Key` and a [dry-run](/idempotency), so a
network blip never double-posts or double-spends:

<CodeGroup>
```ts @postrun/js
import { randomUUID } from "node:crypto";
import { postsCreate } from "@postrun/js";

await postsCreate({
  client: postrun,
  headers: { "Idempotency-Key": randomUUID() },
  body: { dry_run: true, /* ... */ },
});
```

```bash cURL
curl https://api.postrun.ai/v1/posts \
  -H "Authorization: Bearer $POSTRUN_SECRET_KEY" \
  -H "Idempotency-Key: $(uuidgen)" \
  -H "Content-Type: application/json" \
  -d '{ "dry_run": true, "...": "..." }'
```
</CodeGroup>
