Build your entire social and ads product in the browser. Posting, scheduling,
media uploads, account connections, ads insights — all from typed React hooks.
**Your only backend is one endpoint that mints a token.**

Normally this is mostly backend work: an OAuth token vault, a proxy for every
platform's API, chunked media uploads, status polling, and all the fetching and
caching in between. `@postrun/react` removes it. Postrun is the storage, the auth
layer, and the integration to every platform — so a composer, content calendar,
media library, or ads dashboard becomes a set of hooks instead of a service you
run.

## What you get

- **Ship without building a backend.** No OAuth token vault, no per-platform proxy,
  no sync jobs — connect accounts, upload media, publish, schedule, and read
  insights straight from the browser.
- **Write components, not plumbing.** Fetching, caching, revalidation, pagination,
  and live publish-status updates are handled for you. You render state; the hooks
  keep it fresh.
- **Type-safe end to end.** Hooks are fully typed against types generated from the
  API, so a wrong field is a red squiggle in your editor, not a 422 in production.
- **The hard parts are already solved.** A 4 GB video upload that resumes after a
  dropped connection, one-click account connect in a popup, a live post preview —
  each is one hook, not a week of work.
- **Your secret never ships.** The browser only ever holds a short-lived, scoped
  token — never your secret key.

## How it works

`@postrun/react` is three layers on top of a frontend-safe token. You install one
package; they compose automatically.

<Steps>
  <Step title="A frontend-safe token">
    Your backend mints a short-lived, **scoped** token from your `pr_` secret key —
    the secret never reaches the browser. The token's scopes decide exactly what
    the frontend is allowed to do, which is what makes calling Postrun directly
    from the browser safe. See [Authentication](/authentication).
  </Step>
  <Step title="@postrun/js — the typed client">
    Generated from the API's OpenAPI spec: one named type per resource, every call
    typed. It handles auth and requests, plus the fiddly parts — multipart upload
    and payload assembly.
  </Step>
  <Step title="TanStack Query — the data layer">
    Caching, deduplication, revalidation, infinite pagination, and live polling
    while a post publishes. It's bundled and pre-wired in a private client — you
    don't install or configure it.
  </Step>
  <Step title="Hooks — what you call">
    Task-shaped hooks per resource — `useCreatePost`, `useMediaUpload`,
    `useConnect`, `useCalendar`, and the rest — that wrap the client in Query and
    add the orchestration (chunked uploads, the OAuth popup, live status). This is
    the only layer your code touches.
  </Step>
</Steps>

That's the whole reason it works: the token makes direct-from-browser calls safe,
and the layers beneath your hook handle everything between your component and the
platform — so the only code you write is the hook call.

## Install

```bash
npm install @postrun/react
```

Requires React 18.2 or 19. TanStack Query comes bundled — you don't install or
configure it.

## Get a token

Your backend mints the scoped token from a `pr_` secret key and exposes it at an
endpoint your frontend can call. See [Authentication](/authentication) for the
full model and the mint call.

Give the provider a `getToken` that fetches a token from your backend. The
provider caches it and refreshes on a `401`, so this is just a fetch — no caching
or expiry logic of your own:

```ts
async function getToken() {
  const res = await fetch("/api/postrun-token");
  const { token } = await res.json();
  return token;
}
```

## Wrap your app

Mount `<PostrunProvider>` once, near the root, and hand it `getToken`:

```tsx
import { PostrunProvider } from "@postrun/react";

export function Providers({ children }) {
  return <PostrunProvider getToken={getToken}>{children}</PostrunProvider>;
}
```

<Note>
  The provider creates its own private `QueryClient`. You don't add a
  `QueryClientProvider`, and it stays isolated from any TanStack Query setup your
  app already runs. To share one (for devtools or custom defaults), pass
  `queryClient`; to point at another environment, pass `baseUrl`.
</Note>

## Make your first call

Drop a hook into any component under the provider. This one lists the account's
profiles:

```tsx
import { useProfilesInfinite } from "@postrun/react";

function Profiles() {
  const { items, isLoading } = useProfilesInfinite();

  if (isLoading) return <p>Loading…</p>;

  return (
    <ul>
      {items.map((profile) => (
        <li key={profile.id}>{profile.name}</li>
      ))}
    </ul>
  );
}
```

If your profiles render, you're set up — the provider fetched a token, called the
API, and cached the result. No backend route, no token plumbing in your component:
that's the whole model. Every other hook works the same way.
