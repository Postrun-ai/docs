## List a profile's connections

`useConnections` returns a profile's connections, newest first. Filter by `kind`
(`posting` / `ads`) or `status`:

```tsx @postrun/react icon="react"
import { useConnections } from "@postrun/react";

function Connections({ profileId }) {
  const { data } = useConnections(profileId, { kind: "posting" });

  return (
    <ul>
      {data?.data.map((c) => (
        <li key={c.id}>
          {c.avatar_url && <img src={c.avatar_url} width={20} />}
          {c.username ?? c.external_account_name} — {c.status}
        </li>
      ))}
    </ul>
  );
}
```

## Re-auth

When `status === 'needs_reauth'`, the platform's grant expired or was revoked. Run the
same connect flow again for the same `profileId` and `platform` — `useConnect` refreshes
the grant in place. The same connection returns to `active`; no new connection is
created. The `reauth_at` field records when it went stale.

```tsx @postrun/react icon="react"
import { useConnect } from "@postrun/react";

function ReauthButton({ connection }) {
  const { start } = useConnect({
    profileId: connection.profile_id,
    platform: connection.platform,
  });

  if (connection.status !== "needs_reauth") return null;

  return <button onClick={start}>Reconnect {connection.platform}</button>;
}
```

## Switch the bound account

To point a connection at a **different** account on the same grant — say, a different
Facebook Page — read the reachable accounts and select a new one. Unlike the initial
[account selector](/connections/connecting#the-account-selector), this doesn't re-run
OAuth: `useDiscoverableAccounts` reads the accounts already reachable on the stored grant
and `useSelectAccount` rebinds the connection without opening a popup.

```tsx @postrun/react icon="react"
import { useDiscoverableAccounts, useSelectAccount } from "@postrun/react";

function SwitchAccount({ connectionId }) {
  const { data: accounts } = useDiscoverableAccounts(connectionId);
  const select = useSelectAccount();

  return (
    <ul>
      {accounts?.data.map((a) => (
        <li key={a.external_account_id}>
          <button
            onClick={() => select.mutate({ id: connectionId, external_account_id: a.external_account_id })}
          >
            {a.name ?? a.external_account_id}
          </button>
        </li>
      ))}
    </ul>
  );
}
```

## Disconnect

`useDisconnect` removes a connection:

```tsx @postrun/react icon="react"
import { useDisconnect } from "@postrun/react";

const disconnect = useDisconnect();
disconnect.mutate(connectionId);
```

Disconnecting never blocks on scheduled posts. A scheduled post that referenced the
connection will fail at publish time — you'll see it in that post's result. Reconnecting
creates a new connection with a new id; reassign affected posts to it if needed.

## Without the React SDK

The REST equivalents:

- **List** — `GET /v1/profiles/{id}/connections`
- **Get one** — `GET /v1/connections/{id}`
- **Switch account** — `PATCH /v1/connections/{id}` with body `{ "external_account_id": "..." }`
- **Disconnect** — `DELETE /v1/connections/{id}`
