---
title: "Connecting an account"
description: "Run the whole OAuth flow inside your app with one hook — open the popup, pick an account, land an active connection. No redirect, no backend."
---

`useConnect` runs the entire connect flow in your frontend: it opens the platform's
OAuth popup, handles the **grant** (the OAuth authorization the platform gives), shows
an account picker when there's more than one reachable account, and gives you back a
**bound** connection (an account attached to it). You drive your UI off a single
`state.phase`.

## Basic usage

```tsx @postrun/react icon="react"
import { useConnect } from "@postrun/react";

function ConnectButton({ profileId }) {
  const { state, start, select } = useConnect({
    profileId,
    platform: "facebook_page",
    onConnected: (connection) => console.log("connected:", connection.external_account_name),
  });

  // One Facebook login can manage several Pages — let the user pick which one.
  if (state.phase === "picking") {
    return (
      <ul>
        {state.accounts.map((account) => (
          <li key={account.external_account_id}>
            <button onClick={() => select(account.external_account_id)}>
              {account.name ?? account.username ?? account.external_account_id}
            </button>
          </li>
        ))}
      </ul>
    );
  }

  return (
    <button onClick={start} disabled={state.phase === "connecting"}>
      {state.phase === "connecting" ? "Connecting…" : "Connect a Facebook Page"}
    </button>
  );
}
```

## The phases

Everything you render keys off `state.phase`:

| `phase` | What it means |
| --- | --- |
| `preparing` | Minting the connect session. |
| `idle` | Ready — a click will open the popup. |
| `connecting` | The popup is open and the grant is in progress. |
| `picking` | The grant reached multiple accounts — the user must choose. |
| `active` | Done. `state.connection` is the bound connection. |
| `connected_pending` | Granted, but the account binds out-of-band (a slow webhook). A success state, not an error. |
| `cancelled` | The user closed the popup. |
| `error` | The attempt failed; `state.reason` says why. |

When `phase === 'picking'`, render your account picker using `state.accounts` — see
[The account selector](#the-account-selector) below.

On `'error'` or `'cancelled'`, call `reset()` to return to `idle` and offer a retry:

```tsx @postrun/react icon="react"
const { state, start, reset } = useConnect({ profileId, platform: "x" });

if (state.phase === "error" || state.phase === "cancelled") {
  return (
    <div>
      {state.phase === "error" && <p>{state.reason}</p>}
      <button onClick={reset}>Try again</button>
    </div>
  );
}
```

## Error reasons

`state.reason` (also passed to `onError(reason)`) is one of:

| Reason | When it fires |
| --- | --- |
| `prepare_failed` | Couldn't mint the connect session — before any popup opens. |
| `popup_blocked` | The browser blocked the OAuth popup (user must allow popups). |
| `auth_failed` | The OAuth grant itself failed. |
| `connection_not_found` | Couldn't read back the connection after the grant completed. |
| `select_failed` | Account discovery or selection failed. |
| `reauth_required` | The grant must be reconnected before an account can be bound. |

## The account selector

**One OAuth grant can reach more than one account** — a Facebook login manages every
Page you admin, a Google login has every ad account you can access. After the grant
Postrun hands you the list and waits.

- When `state.phase === 'picking'`, `state.accounts` holds the reachable accounts as
  `DiscoverableAccount[]`, each with `external_account_id`, `name`, `username`,
  `avatar_url`, and (for ad accounts) `currency`.
- Call **`select(account.external_account_id)`** to bind the connection to that account
  and move it to `active`.
- If the grant reaches **exactly one** account there's no `picking` phase — Postrun
  binds it automatically.

Personal X or LinkedIn connects usually skip the picker; Facebook Pages and ad accounts
almost always show it.

## One rule: start in the click handler

Call `start()` **directly inside the user's click handler**, with no `await` before
it. Browsers only open a popup synchronously, inside a real user gesture.

For a **single** "Connect X" button, that's automatic: the hook pre-mints the session
on mount (`prepareOnMount` defaults to `true`), so the first click already has a
session.

For a **multi-platform picker** — a row of "Connect X / LinkedIn / TikTok" buttons —
pre-minting every session on mount is wasteful. Set `prepareOnMount: false` and call
`prepare()` on `onPointerEnter`/`onFocus` so the session is ready by the time the
user clicks:

```tsx @postrun/react icon="react"
import { useConnect } from "@postrun/react";

function PlatformRow({ profileId }) {
  const x = useConnect({ profileId, platform: "x", prepareOnMount: false });
  const linkedin = useConnect({ profileId, platform: "linkedin", prepareOnMount: false });

  return (
    <div>
      <button
        onPointerEnter={() => x.prepare()}
        onFocus={() => x.prepare()}
        onClick={x.start}
      >
        Connect X
      </button>
      <button
        onPointerEnter={() => linkedin.prepare()}
        onFocus={() => linkedin.prepare()}
        onClick={linkedin.start}
      >
        Connect LinkedIn
      </button>
    </div>
  );
}
```

## Callbacks

Pass these instead of polling `state` if you'd rather react to events:

- **`onConnected(connection)`** — fired when a connection becomes `active`, with the
  bound connection.
- **`onSuccess()`** — fired on `active` **or** `connected_pending` (use it to close
  your dialog; the connections list auto-refetches either way).
- **`onCancelled()`** — the user closed the popup.
- **`onError(reason)`** — the attempt failed, with the same typed reason as `state.reason`.

## Without the React SDK

`POST /v1/profiles/{id}/connect` returns a `hosted_connect_url` — a Postrun-hosted
page that runs the grant-and-pick flow, then returns the user to your app.

```ts
const res = await fetch(`https://api.postrun.ai/v1/profiles/${profileId}/connect`, {
  method: "POST",
  headers: {
    Authorization: `Bearer ${apiKey}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ platform: "facebook_page" }),
});
const { hosted_connect_url } = await res.json();
window.location.href = hosted_connect_url; // or window.open(hosted_connect_url)
```
