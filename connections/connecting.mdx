---
title: "Connecting an account"
description: "Run the whole OAuth flow inside your app with one hook — open the popup, pick an account, land an active connection. No redirect, no backend."
---

`useConnect` runs the entire connect flow in your frontend: it opens the platform's
OAuth in a popup, handles the grant, shows an account picker when there's more than
one account, and gives you back an active connection. You drive your UI off a single
`state.phase`.

## A connect button

```tsx @postrun/react icon="react"
import { useConnect } from "@postrun/react";

function ConnectButton({ profileId }) {
  const { state, start, select } = useConnect({
    profileId,
    platform: "facebook_page",
    onConnected: (connection) => console.log("connected:", connection.external_account_name),
  });

  // One Facebook login can manage several Pages — let the user pick which one.
  if (state.phase === "picking") {
    return (
      <ul>
        {state.accounts.map((account) => (
          <li key={account.external_account_id}>
            <button onClick={() => select(account.external_account_id)}>
              {account.name ?? account.username ?? account.external_account_id}
            </button>
          </li>
        ))}
      </ul>
    );
  }

  return (
    <button onClick={start} disabled={state.phase === "connecting"}>
      {state.phase === "connecting" ? "Connecting…" : "Connect a Facebook Page"}
    </button>
  );
}
```

## The phases

Everything you render keys off `state.phase`:

| `phase` | What it means | What to render |
| --- | --- | --- |
| `preparing` | Minting the connect session. | A disabled button. |
| `idle` | Ready — a click will open the popup. | Your "Connect" button. |
| `connecting` | The popup is open and the grant is in progress. | A loading state. |
| `picking` | The grant reached **multiple accounts** — the user must choose. | The account picker (see below). |
| `active` | Done. `state.connection` is the bound connection. | A success state. |
| `connected_pending` | Granted, but the account binds out-of-band (a slow webhook). A **success** state, not an error. | "Almost there" — refetch the list. |
| `cancelled` | The user closed the popup. | Offer a retry (`reset()`). |
| `error` | The attempt failed; `state.reason` says why. | An error state with a retry. |

## The account selector

This is the step that trips people up, so it's worth being explicit. **One OAuth
grant can reach more than one account** — a Facebook login manages every Page you
admin, a Google login has every ad account you can access. Postrun can't guess which
one you mean, so after the grant it hands you the list and waits for you to pick.

- When `state.phase` is `picking`, `state.accounts` holds the reachable accounts —
  each a `DiscoverableAccount` with `external_account_id`, `name`, `username`,
  `avatar_url`, and (for ad accounts) `currency`.
- Render them, and call **`select(account.external_account_id)`** with the chosen one.
  That binds the connection to that account and moves it to `active`.
- If the grant reaches **exactly one** account, there's no `picking` phase — Postrun
  binds it automatically and you go straight to `active`.

So a personal X or LinkedIn connect usually skips the picker entirely, while Facebook
Pages and ad accounts almost always show it.

## One rule: start in the click handler

Call `start()` **directly inside the user's click handler**, with no `await` before
it. Browsers only let you open a popup synchronously, inside a real user gesture — so
`start()` opens the OAuth popup immediately, then does its async work.

For a **single** "Connect X" button, that's automatic: the hook pre-mints the session
on mount (`prepareOnMount` defaults to `true`), so the first click already has a
session and opens the popup.

For a **multi-platform picker** (a row of "Connect X / LinkedIn / TikTok" buttons),
pre-minting every session on mount is wasteful. Set `prepareOnMount: false` and call
`prepare()` on each button's `onPointerEnter` / `onFocus`, so the session is ready by
the time the user clicks — without minting all of them up front.

## Callbacks

Pass these instead of polling `state` if you'd rather react to events:

- **`onConnected(connection)`** — fired when a connection becomes `active`, with the
  bound connection.
- **`onSuccess()`** — fired on `active` **or** `connected_pending` (use it to close
  your dialog; the connections list auto-refetches either way).
- **`onCancelled()`** — the user closed the popup.
- **`onError(reason)`** — the attempt failed, with a typed reason.

## No-JS fallback

If you can't use the React SDK, `POST /v1/profiles/{id}/connect` returns a
`hosted_connect_url` — a Postrun-hosted page that runs the exact same grant-and-pick
flow, then returns the user to you.
