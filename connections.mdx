---
title: "Connections"
description: "A connection links a profile to a platform account — an X handle, a Facebook Page, a Google Ads account — that it can post or advertise through."
---

A profile can have several connections, one per platform account. Postrun holds the
OAuth grant and refreshes it — you never store or rotate platform tokens.

These examples assume the React SDK is set up — see [Getting started with React](/react/get-started).

## Status

`status` is the field you branch on — it tells you whether a connection can be used:

| Status | Meaning |
| --- | --- |
| `pending` | Access was granted, but no account is bound yet. The user still needs to [pick which account](/connections/connecting#the-account-selector) — one login can reach several Pages or ad accounts. |
| `active` | Bound to an account and ready to post or run ads. |
| `needs_reauth` | The platform's grant expired or was revoked. Reconnect the account to fix it. |

## Fields

| Field | Meaning |
| --- | --- |
| `platform` | `x`, `linkedin`, `instagram`, `facebook_page`, `tiktok` (social) or `google_ads`, `meta_ads` (ads). |
| `kind` | `posting` or `ads` — derived from the platform, never set directly. |
| `external_account_id` / `external_account_name` | The platform account this connection is bound to. Null while `status` is `pending`. |
| `username` / `avatar_url` / `profile_url` | Cached display fields for social accounts. Null for ad accounts — use `external_account_name` instead. |
| `currency` | The account's currency (ad accounts only; null for social). |
| `reauth_at` | When the grant went stale (set when `status` is `needs_reauth`). |
| `profile_id` | The profile it belongs to. |
| `created_at` / `updated_at` | Timestamps. |
