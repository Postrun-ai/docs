---
title: "Media"
description: "Upload a file once; Postrun renders a ready rendition per platform and returns a typed verdict for each."
---

Upload a file (or import via `source_url`), name your `targets`, and Postrun produces a ready rendition for each — transcoding only as needed and validating against each platform's current limits. A file that can't fit a platform surfaces a typed error in `per_platform[target].errors` before you publish; Postrun adjusts the technical envelope (format, codec, size) but never crops or trims your content.

<CodeGroup>
```tsx @postrun/react icon="react"
import { useMediaUpload } from "@postrun/react";

function Uploader({ profileId }) {
  const { add } = useMediaUpload();

  async function upload(file: File) {
    const [asset] = await add(file, { profileId, targets: ["x", "instagram"] });

    const x = asset.per_platform.x;
    // x.status   → "ready" or "failed"
    // x.url      → the rendition URL (null when failed)
    // x.errors   → blocking findings (e.g. aspect ratio out of range)
    // x.warnings → non-blocking adjustments (e.g. re-encoded to H.264)
  }
}
```

```ts @postrun/js icon="square-js"
import { mediaCreate, waitForMedia } from "@postrun/js";

const { data: media } = await mediaCreate({
  client: postrun,
  body: { profile_id: "prof_3b9d6bcd", targets: ["x", "instagram"] },
});

const asset = await waitForMedia(media.id, { client: postrun });

const x = asset.per_platform.x;
// x.status   → "ready" or "failed"
// x.url      → the rendition URL (null when failed)
// x.errors   → blocking findings (e.g. aspect ratio out of range)
// x.warnings → non-blocking adjustments (e.g. re-encoded to H.264)
```
